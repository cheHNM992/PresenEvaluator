# プレゼン評価システム仕様書

## 1. システム概要

プレゼンテーションの録音と資料を分析し、評価基準に基づいて採点およびアドバイスを提供するシステム。

## 2. 入力データ

### 2.1 音声データ
- フォーマット: MP3/WAV
- 長さ: 最大15分
- 保存場所: PC内のローカルフォルダ

### 2.2 プレゼン資料
- フォーマット: PowerPoint
- スライド数: 最大30枚
- 保存場所: PC内のローカルフォルダ

## 3. 処理フロー

1. 音声の文字起こし処理
2. 資料の内容抽出
3. 評価基準に基づく分析
4. スコア計算と改善点の特定
5. 結果の表示

## 4. 分析コンポーネント

### 4.1 音声分析モジュール
- **機能**: 音声ファイルを文字起こし
- **使用技術**: OpenAI Whisper API
- **分析項目**:
  - 話速（WPM: Words Per Minute）
  - 間の置き方
  - 音量/声の大きさの変化
  - 感情表現
  - フィラーワード（「えーと」「あの」など）の使用頻度

### 4.2 資料分析モジュール
- **機能**: PowerPointからテキストと構造を抽出
- **使用技術**: python-pptx など
- **分析項目**:
  - スライド当たりの文字数
  - 図表の数と配置
  - 構成（導入、本論、結論の明確さ）
  - フォントサイズと色使い

## 5. 評価基準

以下の項目を5段階（1-5）で評価：

1. **内容（30%）**
   - トピックの明確さ
   - 論理展開
   - データ/例の適切さ

2. **プレゼン技術（30%）**
   - 話し方（流暢さ、明瞭さ）
   - 声のトーン変化
   - アイコンタクト（音声データから推測）

3. **視覚資料（20%）**
   - デザインの質
   - 情報の適切な量
   - 視覚的な補助効果

4. **構成（20%）**
   - 導入部の効果
   - 本論の組立て
   - 結論のインパクト

## 6. アドバイス生成

- 各評価項目の低いスコアに対応するアドバイスをテンプレート化
- 音声と資料の相関性に基づく総合アドバイス
- 3つの具体的な改善点を提案

## 7. 出力形式

### 7.1 数値評価
- 各項目のスコア（1-5）
- 重み付け計算による総合スコア（100点満点）

### 7.2 テキスト評価
- 長所（3点）
- 改善点（3点）
- 具体的なアドバイス（各項目ごと）

## 8. 技術スタック

- **プログラミング言語**: Python
- **音声処理**: Whisper API
- **ドキュメント処理**: python-pptx
- **テキスト分析**: NLTK または spaCy (シンプルな感情分析)
